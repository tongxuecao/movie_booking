# 电影票预订系统 — 课设项目规划

> 目标：一个月内完成，本地单机环境交付，答辩高分

## 🎯 战略定位

在有限时间内，核心目标是：**在本地搭建出一套能够"理论上抗住万级并发"的系统骨架，并在答辩中展现出技术亮点。**

### 保留的亮点（答辩加分项）

| 亮点 | 说明 |
|------|------|
| Vue 3 + Vant 4 选座界面 | 响应式设计，丝滑的座位选择体验 |
| Redis Lua 脚本原子锁座 | 抗住万级并发的核心底牌 |
| Redis Stream 异步下单 | 流量削峰，体现架构思维 |
| 虚拟钱包支付 | 乐观锁扣减余额，注册送 1000 |

### 断舍离（果断砍掉）

- **真实云服务器**：全部在本地完成
- **微服务架构**：单体 Spring Boot 即可
- **真实支付**：用虚拟钱包替代

## 🏗️ 技术架构

```
前端 (Vue 3 + Vant 4)
    ↓ HTTP API
后端 (Spring Boot 3.2.5 单体)
    ├── MySQL 8.0 (持久化)
    ├── Redis (锁座 + Stream)
    └── JWT (认证)
```

### 核心流程

```
选座 → Redis Lua 原子锁座 (15分钟TTL)
    → 创建订单 → Redis Stream 异步处理
    → 虚拟钱包支付 (乐观锁)
    → 前端轮询订单状态
```

## 💡 答辩话术

> "本系统在本地单机环境下进行了全链路模拟，架构设计充分考虑了万级并发场景。
>
> 系统采用四层防护：前端实现按钮防抖；核心层采用 **Redis + Lua 脚本**在内存中原子锁座，将读写性能提升数倍；同时引入 **Redis Stream** 实现订单异步削峰，确保数据库不被瞬时流量冲垮。
>
> 经 JMeter 压测验证，系统在并发下响应时间低于 50ms，且座位零超卖。"
