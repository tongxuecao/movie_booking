# 订单系统流程图

## 完整流程

```mermaid
graph TD
    A([用户进入]) --> B[挑选电影场次座位]
    B --> C[点击提交购票]

    C --> D{Redis锁座}
    D -->|失败| E[提示座位已被抢占]
    E --> B

    D -->|成功| F[更新数据库座位状态locked]
    F --> G[创建订单pending]
    G --> H[设置15分钟超时]
    H --> I[推入Redis队列异步处理]

    I --> J{余额校验}
    J -->|余额不足| K[保持订单pending]
    K --> L{用户选择}
    L -->|立即充值| M[充值成功]
    M --> N[手动支付订单]
    L -->|稍后支付| O[进入订单列表等待]

    J -->|余额充足| P[乐观锁扣款]
    P -->|扣款失败| Q[订单cancelled释放锁]
    Q --> R([结束失败])

    P -->|扣款成功| S[订单paid]
    S --> T[座位状态sold]
    T --> U[生成电子取票码]
    U --> V([结束成功])

    N --> P

    H --> W{定时扫描}
    W -->|超时| X[订单cancelled]
    X --> Y[释放锁恢复座位]
    Y --> Z([结束超时])
```

## 状态说明

| 状态 | 含义 | 触发条件 |
|------|------|----------|
| pending | 待支付 | 订单创建后 |
| paid | 已支付 | 支付成功 |
| cancelled | 已取消 | 超时或支付失败 |
| refunded | 已退款 | 用户退票 |

## 核心机制

1. **锁座**：Redis Lua 原子操作，15分钟TTL
2. **异步下单**：Redis Stream 削峰，乐观锁扣款
3. **超时取消**：定时任务扫描加Redis过期键双重保障
4. **座位持久化**：数据库记录locked和sold状态
5. **取票码**：支付成功后生成TK加8位随机码
