# 电影票预订系统 — 功能模块结构（合并后）

## 合并方案

系统按业务内聚性划分为 **6大模块**：用户端3个 + 管理端3个。

```
合并前（13个）                    合并后（6个）
─────────────────────────────────────────────────
【用户端】
① 用户模块         ─┐
                    ├─ 用户模块（认证+资料+钱包）
                    │
② 电影浏览模块     ─┐
③ 影院浏览模块     ─┤
⑦ 评价模块         ─┼─ 电影模块（浏览+影院+评价+想看）
                    │
④ 选座购票模块     ─┐
⑤ 支付模块         ─┼─ 订单模块（选座→锁座→支付→订单）
⑥ 订单模块         ─┘

【管理端】
⑧ 电影管理         ─┐
⑨ 排片管理         ─┼─ 内容管理模块（电影+排片+影院+影厅）
⑩ 影院管理         ─┘

⑪ 订单管理         ─── 订单管理模块

⑫ 用户管理         ─┐
⑬ 数据统计         ─┼─ 系统管理模块（用户管理+统计）
```

---

## 合并后6大模块详情

### 用户端（3个模块）

| 模块 | 功能项 | 描述 |
|------|--------|------|
| **1. 用户模块** | 注册 | 填写用户名、手机号、密码完成注册，注册即送1000元虚拟钱包余额 |
| | 登录 | 用户名+密码登录，登录后获取JWT Token（7天有效） |
| | 个人中心 | 查看/修改个人资料（头像、昵称、手机号），修改密码 |
| | 钱包充值 | 输入充值金额和支付密码，虚拟钱包充值 |
| **2. 电影模块** | 电影列表 | 分页展示电影，支持"正在热映"/"即将上映"标签切换，关键词搜索 |
| | 电影详情 | 查看电影海报图集、基本信息、剧情简介、用户评分 |
| | 票房排行 | 首页侧边栏展示今日票房排行榜和近期最受期待电影 |
| | 想看功能 | 用户标记想看的电影，排行榜实时更新（Redis Set） |
| | 影院列表 | 分页展示影院，支持名称/地址搜索 |
| | 影院详情 | 查看影院基本信息及该影院所有电影的排片场次 |
| | 电影评价 | 1-10分打分，可选500字文字评价，分页查看评价列表 |
| **3. 订单模块** | 场次查询 | 按电影/影院/日期查询场次，按影院分组展示 |
| | 在线选座 | 可视化座位网格，实时显示已售/可选状态，支持普通座/VIP座/情侣座 |
| | Redis锁座 | 调用Redis Lua脚本原子锁座（15分钟TTL），防止超售 |
| | 创建订单 | 锁座成功后创建待支付订单，生成唯一订单号 |
| | 虚拟支付 | 输入支付密码完成钱包支付（乐观锁扣款），余额不足时可充值重试 |
| | 状态轮询 | 前端每2秒轮询订单状态（最多20次） |
| | 超时处理 | 15分钟内未支付自动取消订单并释放Rdis座位锁 |
| | 订单管理 | 查看全部/已支付/待支付/已取消订单列表，查看详情，取消退款 |

### 管理端（3个模块）

| 模块 | 功能项 | 描述 |
|------|--------|------|
| **4. 内容管理模块** | 电影CRUD | 新增/编辑/删除电影，支持海报上传 |
| | 影院CRUD | 新增/编辑/删除影院信息（名称、地址、电话、状态） |
| | 影厅管理 | 在影院下新增/编辑/删除影厅，设置座位行数/列数/影厅类型 |
| | 场次管理 | 新增/删除场次（级联选择电影→影院→影厅），支持批量排片 |
| **5. 订单管理模块** | 订单查看 | 查看所有用户订单列表，按支付状态筛选，查看订单详情 |
| **6. 系统管理模块** | 用户管理 | 查看/搜索用户列表，编辑用户信息（用户名/手机号/角色/钱包余额），启用/禁用用户 |
| | 数据统计 | 仪表盘：今日订单数+营业额、7天趋势双Y轴折线图、票房TOP10柱状图、想看TOP10柱状图 |

## 合并逻辑说明

| 合并后模块 | 包含原模块 | 合并理由 |
|-----------|-----------|---------|
| **用户模块** | 用户模块 | 注册登录、个人资料、钱包充值是用户身份相关基础功能，不可拆分 |
| **电影模块** | 电影浏览 + 影院浏览 + 评价 | 都是"电影信息消费"场景，用户先浏览信息再决定买票，内聚性强 |
| **订单模块** | 选座购票 + 支付 + 订单 | 选座→锁座→支付→订单管理是**同一条业务流程**，拆开反而割裂购票闭环 |
| **内容管理模块** | 电影管理 + 排片管理 + 影院管理 | 管理端对业务资源（电影/影院/影厅/场次）的CRUD，管理模式一致 |
| **订单管理模块** | 订单管理(管理端) | 管理端查看所有订单，与用户端订单逻辑对应但视角不同 |
| **系统管理模块** | 用户管理 + 数据统计 | 系统级运营功能，用户管理和数据分析放在一起 |
